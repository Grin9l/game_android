# Как проверить логи ошибок Android

## Способ 1: Через приложение на телефоне (САМЫЙ ПРОСТОЙ)

### Шаг 1: Установите приложение для просмотра логов

**Вариант A: Log Viewer (рекомендуется)**
1. Откройте Google Play Store на телефоне
2. Найдите приложение "Log Viewer" или "aLogcat"
3. Установите его

**Вариант B: MatLog (бесплатный, открытый исходный код)**
1. Найдите "MatLog" в Google Play
2. Или скачайте с GitHub: https://github.com/plusCubed/matlog/releases

### Шаг 2: Просмотр логов

1. Откройте установленное приложение
2. Нажмите кнопку "Start" или "Record" для начала записи логов
3. Запустите игру "Ксюша Единорог"
4. Вернитесь в приложение для логов
5. Найдите записи с тегом "GameView" или "MainActivity"
6. Скопируйте или сделайте скриншот ошибок

## Способ 2: Через ADB (если телефон подключен к компьютеру)

### Что такое ADB?

ADB (Android Debug Bridge) - это инструмент командной строки от Google для отладки Android-устройств.

### Шаг 1: Установка ADB

**Windows:**
1. Скачайте "Platform Tools" с официального сайта:
   https://developer.android.com/studio/releases/platform-tools
2. Распакуйте архив (например, в `C:\adb`)
3. Добавьте папку в PATH (опционально, но удобнее)

**Или используйте Android Studio:**
- ADB уже включен в Android Studio
- Обычно находится в: `C:\Users\ВашеИмя\AppData\Local\Android\Sdk\platform-tools\`

### Шаг 2: Включите отладку по USB на телефоне

1. Откройте **Настройки** → **О телефоне**
2. Нажмите 7 раз на **Номер сборки** (включит режим разработчика)
3. Вернитесь в **Настройки** → **Для разработчиков**
4. Включите **Отладка по USB**

### Шаг 3: Подключите телефон к компьютеру

1. Подключите телефон через USB кабель
2. На телефоне появится запрос "Разрешить отладку по USB?" → нажмите **Разрешить**

### Шаг 4: Проверьте подключение

Откройте командную строку (PowerShell) и выполните:

```powershell
# Если ADB в PATH:
adb devices

# Если ADB не в PATH, перейдите в папку с ADB:
cd C:\adb
.\adb.exe devices
```

Должно показать что-то вроде:
```
List of devices attached
ABC123XYZ    device
```

### Шаг 5: Просмотр логов

```powershell
# Все логи с тегом GameView или MainActivity:
adb logcat | Select-String "GameView|MainActivity"

# Или все ошибки:
adb logcat *:E

# Или сохранить в файл:
adb logcat > logs.txt
```

## Способ 3: Через Android Studio (если установлен)

1. Откройте Android Studio
2. Подключите телефон через USB
3. Внизу Android Studio откройте вкладку **Logcat**
4. Запустите игру на телефоне
5. В Logcat будут видны все логи

---

## Способ 4: Через беспроводное подключение ADB (без USB)

Если не хотите использовать USB кабель:

1. Подключите телефон и компьютер к одной Wi-Fi сети
2. На телефоне включите "Отладка по USB" (см. выше)
3. В Android Studio: **Tools** → **Wireless Debugging**
4. Или через ADB:
   ```powershell
   adb connect IP_АДРЕС_ТЕЛЕФОНА:5555
   ```

---

## Что искать в логах?

Ищите строки с:
- `GameView` - логи из игрового view
- `MainActivity` - логи из главной активности
- `ERROR` или `Exception` - ошибки
- `FATAL` - критические ошибки

**Пример хорошего лога:**
```
D/GameView: GameView created
D/GameView: surfaceCreated
D/GameView: Screen size: 1080x1920
D/GameView: startGame
D/GameView: GameThread started
```

**Пример плохого лога (ошибка):**
```
E/GameView: Error in game loop
E/AndroidRuntime: FATAL EXCEPTION: GameThread
```

## Рекомендация

**Для начинающих:** Используйте **Способ 1** (приложение на телефоне) - это самый простой способ, не требует компьютера и настройки.

**Для продвинутых:** Используйте **Способ 2** (ADB) - дает больше контроля и возможностей.

---

## Если ничего не помогает

1. Перезагрузите телефон
2. Удалите и переустановите приложение
3. Проверьте версию Android (нужна 5.0+)
4. Попробуйте на другом телефоне
